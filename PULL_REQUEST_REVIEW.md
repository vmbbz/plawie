# Maintainer Review & Merge Guide: Local LLM Milestone (Audit/Polish/Stabilization)

## Overview
This PR introduces a production-grade stabilize-and-polish layer for the Local LLM (Ollama) infrastructure. It eliminates the previous memory-related crashes and resolves the 15-second latency issues when using the Gateway with local models.

## Key Changes to Review
1.  **`lib/services/gateway_service.dart`**:
    *   **Context Clamping:** We now force `contextWindow: 4096` into the provider block. This is critical—without this, the gateway requests a 200k buffer and crashes the entire Android app.
    *   **Prompt Override:** The 27k character system prompt is swapped for an optimized ~600-character "Tool Scaffolding" prompt. This reduces Time-To-First-Token (TTFT) from 15s to <500ms while retaining tool-call accuracy.
2.  **`ARCHITECTURE_LOCAL_LLM.md`**:
    *   Audited and updated the facts regarding phone-side compute. This documentation is now the source-of-truth for the "AnyClaw" vs "fllama" debate.
3.  **`local_llm_screen.dart`**:
    *   Added a "High-RAM" toggle for power users with 12GB+ RAM to bypass safety clamps if they demand the full agent toolset.

## Merge & Verification Steps
1.  **Safety Check:** After merging, start the **Integrated Agent Hub** in the Local LLM settings.
2.  **Health Scan:** Ensure the "ONLINE" status badge appears (green).
3.  **Inference Test:** Navigate to the main chat and send: "Tell me your name. Are you OpenClaw?".
    *   *Success:* The model responds instantly (TTFT < 1s).
    *   *Failure:* If it crashes (LMK), check that `PreferencesService().enableFullContext` is set to **OFF**.

## Note on Conflicts
**DO NOT** manually re-add the `'model': model` field to `chat.send` in `gateway_service.dart`. This field specifically triggers a JSON Schema validation error in the Node.js OpenClaw gateway; the model selection is instead handled via stateful disk persistence in `_syncModelToConfig`.
